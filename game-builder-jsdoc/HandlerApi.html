    HandlerApi()

    Main API entry point for message handlers. Exposes all the things user-written handlers are allowed to do. 'getActor' is the main way to modify the handling actor.

    new HandlerApi()

    Members

    dt

    The time, in seconds, since the last Tick. NOTE: Only valid for 'OnTick'!

    memory

    Access (read and write) the handling actor's memories.

    message

    Get the data associated with the message you're handling. The contents of this completely depends on the message.

    properties

    Access your declared properties. For example, if you declared "property Number health", you would access it like: api.properties.health

    props

    Shortcut for 'properties'.

    time

    Get the absolute game time, in seconds.

    getActor() → {HandlingActor}

    This is the main way to access the current actor's properties, such as its position. Because this is the current handling actor, you can modify its properties as well.
    Returns:
    HandlingActor - The current actor that your handler is running for.

    getOtherActor(name) → {OtherActor}

    Use this to read properties of another actor. Remember, you can *only* read other actors, you can not modify them. The main way to interact with other actors is by sending them messages (see sendMessage(...))
    Parameters:
    Name Type Description
    name string The name of the actor you'd like to inspect.
    Returns:
    OtherActor - A read-only interface for the actor, with functions like getPosition()

    isDead() → {boolean}

    Convenience function, only valid if the actor has "Takes damage".
    Returns:
    boolean

    isValidActor(name) → {boolean}

    Like doesActorExist, but also returns 'false' if name is undefined or empty.
    Parameters:
    Name Type Description
    name The name of the actor to check.
    Returns:
    boolean

    overlapSphere(center, radius) → {Array.<string>}

    Find all actors inside a sphere.
    Parameters:

    center THREE.Vector3
    radius number
    Returns:
    Array.<string> - Array of actor names that touch the given sphere.

    raycast(origin, direction, maxDistance) → {Array.<string>}

    Performs a good ol' raycast. Note that this will return all hits along the ray, not just the first one. So it "goes through" actors. So keep 'maxDistance' reasonable.
    Parameters:
    Name Type Description
    origin THREE.Vector3
    direction THREE.Vector3
    maxDistance number
    Returns:
    Array.<string> - Array of actor names that the ray hit.
